Auftakt

Technische Grundlage: Prompten am Beispiel von Bahnwärter Thiel

Merksatz

Rolle + Aufgabe + Kontext + Format

Gute Prompts sind konkret. Gute Ergebnisse entstehen durch gezieltes Nachschärfen. Deshalb arbeiten wir hier mit einem Beispiel aus dem Deutschunterricht.

Prompt-Formel

Rolle Aufgabe Kontext Format

Erst klar fragen – dann gezielt nachschärfen.

Prompt-Impuls 1

Start: Unspezifischer Prompt

Ausgangslage

Ein sehr kurzer Prompt liefert oft nur allgemeine Ergebnisse.

Beispiel:

Erstelle eine Lerneinheit zu Bahnwärter Thiel.

Leitfrage: Was fehlt, damit die Antwort wirklich zum Unterricht passt?

Prompt-Impuls 2

Verbesserung: Rolle + Kontext

Erste Verbesserung

Ein besserer Prompt benennt Rolle, Lernzielgruppe und gewünschte Form.

Beispiel:

Du bist eine Deutschlehrer*in. Erstelle eine interaktive Lerneinheit zu Bahnwärter Thiel für eine 2. Klasse (Sek II).

Leitfrage: Welche Zusatzinfos machen den Auftrag klarer?

Prompt-Impuls 3

Iteration 1: Konkrete Szene

Gezielt nachschärfen

Mit einer konkreten Szene wird die Antwort textnäher und fachlich präziser.

Beispiel:

Beziehe dich konkret auf die Szene mit Tobias’ Tod und formuliere drei textnahe Fragen.

Leitfrage: Welche Textstelle oder Szene soll bearbeitet werden?

Prompt-Impuls 4

Iteration 2: Fachliche Präzisierung

Noch präziser

Die stärksten Ergebnisse entstehen, wenn du das gewünschte Format fachlich genau festlegst.

Beispiel:

Ergänze direkte Zitate mit Seitenangabe und formuliere die Fragen so, dass sie für eine interaktive Lernstrecke geeignet sind.

Leitfrage: Welche Form, Tiefe oder Quellenangaben brauchst du zusätzlich?

Lernroute

Schritt für Schritt durch die GitHub-Lernstrecke

Phase 1

Einstieg und Konto

Schritt 1

GitHub aufrufen und Startpunkt finden

Öffne GitHub im Browser und finde den Einstieg zur Registrierung.

Erfolgskriterium: Du siehst, wo du die Kontoerstellung startest.

Schritt 2

Konto erstellen und Username festhalten

Fülle die Pflichtfelder aus und schliesse die Registrierung ab.

Erfolgskriterium: Dein gewählter Username ist notiert und du kannst dich anmelden.

Schritt 3

Im Dashboard orientieren

Suche nach dem Login das Menü, mit dem du neue Inhalte anlegst.

Erfolgskriterium: Du weisst, wo das Plus-Menü liegt.

Phase 2

Repository aufsetzen

Schritt 4

New repository auswählen

Öffne über das Plus-Menü den Weg zur Erstellung eines neuen Repositories.

Erfolgskriterium: Das Formular für das neue Repository ist geöffnet.

Schritt 5

Repository-Formular verstehen

Lies das Formular genau und entscheide, wie dein Repository heissen soll.

Erfolgskriterium: Name und Sichtbarkeit sind bewusst gewählt.

Schritt 6

Formular ausfüllen und Repository erstellen

Trage den Namen ein und erstelle das Repository.

Erfolgskriterium: Dein Repository existiert und ist geöffnet.

Phase 3

Dateien hochladen und veröffentlichen

Schritt 7

Neues Repository prüfen

Kontrolliere, ob dein Repository erfolgreich angelegt wurde.

Erfolgskriterium: Du erkennst den Link zum Hochladen vorhandener Dateien.

Schritt 8

Dateien hochladen vorbereiten

Lade die Webdateien hoch oder wähle sie über den Upload-Dialog aus.

Erfolgskriterium: Die Dateien sind bereit für den Commit.

Schritt 9

Dateien vom Computer auswählen

Wähle die richtigen Dateien aus deinem Projektordner aus.

Erfolgskriterium: Du lädst nur die benötigten Dateien hoch.

Schritt 10

Repository-Settings öffnen

Gehe in die Einstellungen deines Repositories.

Erfolgskriterium: Der Bereich für GitHub Pages ist erreichbar.

Phase 4

Pages aktivieren und Ergebnis sichern

Schritt 11

GitHub Pages konfigurieren

Wähle in den Einstellungen den Bereich für GitHub Pages aus.

Erfolgskriterium: Die Publikationsart ist auf einen Branch eingestellt.

Schritt 12

Seite aufrufen, URL eintragen und reflektieren

Speichere die Pages-Einstellungen, warte auf die Bereitstellung und prüfe die fertige Seite.

Erfolgskriterium: Die Website ist über eine URL erreichbar.
